# 3dtile_4x4x7200 stitch + Neuroglancer — troubleshooting reference

Last updated: 2026-05-26 — Allison

## Goal

Stitch the 4×4×7200 pan-ASLM acquisition on Misha, host the stitched
volume on Neuroglancer for PI review.

## What works

- **Stitching pipeline on Misha**: full 4×4×7200 stitched (job 1954258, 13 h)
  using pi2/NRStitcher 4.5 built from source on Misha. All 16 tiles, all
  pairwise displacement fields, all transformations computed cleanly.
  Output: 1.86 TB stitched `.raw`.
- **Precomputed conversion**: 1,790,146 base-resolution chunks + 3 MIPs (×2, ×4, ×8 downsamples) via
  cloud-volume + igneous (job 1958609, 3 h). Chunk count matches expected exactly.
- **Neuroglancer rendering via SSH tunnel + local `serve.py`**: works with
  contrast tuned for actual data range. Volume is browsable; tiles are
  registered; structure visible at appropriate zoom + shader settings.

## Known limitations

- Only **MIPs 1–3 built** (igneous "3 mips per batch" memory limit). MIPs 4–5
  would require another conversion pass; not critical for review since MIP-3
  fits the whole volume comfortably.
- **NG access is currently single-user** (via your laptop's SSH tunnel + `serve.py`
  on Misha login node). PI cannot click a link without that tunnel running.
  Long-term plan: serve the `precomputed/` directory from the kuanlab nginx
  server so the URL is publicly accessible.

## Data is faithful but dim

Source ss_single TIFFs have value range **0–717** (typical max). Stitched
output preserves this exactly (max=433 sampled at z=2480). No data
attenuation in the pipeline. Contrast tuning in Neuroglancer needs an
explicit range, e.g.:

```glsl
#uicontrol invlerp contrast(range=[0, 3000], window=[0, 411])
void main() { emitGrayscale(contrast()); }
```

(Range and window tuned to the actual data percentile.)

## Things that bit us along the way (so PI knows what's been ruled out)

| Symptom | Root cause | Fix |
|---|---|---|
| sbatch refused with "DOS line breaks" | Windows-written file had CRLF | `newline='\n'` on every shell-script write in core.py |
| `ModuleNotFoundError: networkx` inside nr_stitcher | pi2_env missing transitive deps | `pip install` deps from grep of `base.py` imports |
| Conda env appears active but python uses base | `module load miniconda` puts module's bin/ ahead of env's | Removed `module load miniconda` from sbatch; conda init via `source conda.sh` only |
| nr_stitcher `ValueError: setting an array element with a sequence` | numpy 2.x scalar conversion change | `pip install "numpy<2"` (now pinned, do not let upgrade) |
| EmptyVolumeException during MIPs after `Base resolution write complete` printed | tensorstore async writes returned from `.result()` before durable | Switched base write to cloud-volume (synchronous per-chunk fopen/fwrite/fclose) |
| Job OOM at 64GB / 128GB ceiling with sharded tensorstore | Tensorstore buffered in-progress shards unboundedly | Dropped sharding; later switched off tensorstore entirely |
| 85% of chunks silently missing on disk despite tensorstore reporting completion | Tensorstore write cache + memory eviction | Cloud-volume write fixed this; 100% chunk completion verified |
| `HTTP error 0` / `Connection refused` from Neuroglancer | SSH tunnel landed on a different login node than `serve.py` | Run `serve.py` on whichever node tunnel lands on; or always do both in the same `ssh -L` session |
| Volume looks all black in Neuroglancer | Real data range is 0-717 (~1% of uint16 max) | Shader needs explicit range `[0, 500]` or multiplier ~90-130, NOT default 0-65535 |

## How to talk to the data right now

- **In Fiji**: `extract_for_fiji.py` (in the bundle dir) produces `mip3_volume.tif`
  (3.6 GB, whole stitched volume at 1.624 µm/voxel) and `base_zslab.tif`
  (3.4 GB, central 4096×4096×100 at full resolution). scp either to local
  and File → Open in Fiji.
- **In Neuroglancer (via tunnel)**:
  1. `ssh -L 8000:localhost:8000 misha` (keep open)
  2. On Misha: `cd <bundle>`, activate `pi2_env`, `nohup python serve.py --port 8000 &`
  3. Open `https://neuroglancer-demo.appspot.com/`, add source
     `precomputed://http://127.0.0.1:8000/precomputed`
  4. Apply shader with explicit range, e.g. `range=[0, 500]`.
- **Direct chunk inspection (sanity check, Misha)**:
  ```
  python -c "import numpy as np; d=np.fromfile('precomputed/203_203_200/<chunk>',dtype=np.uint16); print(d.min(),d.max())"
  ```
